# Security Tools

Ця папка містить інструменти та конфігурації для забезпечення безпеки під час розробки.

## Gitleaks Setup

Gitleaks - це інструмент для виявлення секретів та конфіденційної інформації у git репозиторіях.

### Файли

-   **`setup-global-gitleaks.sh`** - Скрипт для глобального налаштування gitleaks (рекомендовано)
-   **`install-gitleaks-hooks.sh`** - Скрипт для встановлення gitleaks pre-commit hooks для окремих репозиторіїв
-   **`.gitleaks.toml`** - Конфігураційний файл з правилами для виявлення різних типів секретів
-   **`GITLEAKS_SETUP.md`** - Докладна документація та інструкції з налаштування

### Швидке встановлення (Глобальне)

Для глобального налаштування, щоб gitleaks працював у всіх репозиторіях:

```bash
./Helpers/Security/setup-global-gitleaks.sh
```

### Використання для окремих репозиторіїв

Для налаштування gitleaks для конкретного репозиторію:

```bash
./Helpers/Security/install-gitleaks-hooks.sh
```

### Команди

Після глобального налаштування:

```bash
# Перевірка репозиторію
git gitleaks

# Перевірка staged файлів
git gitleaks --pre-commit

# Сканування папки
gitleaks dir /шлях/до/папки
```

### Що виявляє

-   API ключі
-   Паролі
-   Приватні ключі
-   AWS креденшали
-   GitHub токени
-   Slack токени
-   Загальні секрети

### Переваги глобального налаштування

✅ Автоматично працює у всіх репозиторіях  
✅ Не потребує додаткового налаштування для нових проектів  
✅ Централізована конфігурація  
✅ Глобальні ignore правила

Для детальної інформації дивіться [`GITLEAKS_SETUP.md`](GITLEAKS_SETUP.md).
